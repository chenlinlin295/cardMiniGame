## 1. 基础定义

## 1.1 平台与坐标

- 平台：微信小程序移动端（竖屏）
- 设计基准：`750 x 1334`（@2x）
- 安全区建议：
  - 顶部安全区：`y = 0 ~ 88`
  - 底部安全区：`y = 1230 ~ 1334`
- 位置描述采用“相对屏幕占比 + 建议像素范围”

## 1.2 页面列表（信息架构）

1. 首页 `Home`
2. 对局页 `Battle`
3. 设置弹窗 `SettingsModal`
4. 规则弹窗 `RulesModal`
5. 转场弹层（大批量动物来袭）`InvasionModal`
6. 暂停弹窗 `PauseModal`
7. 复活弹窗 `ReviveModal`
8. 最终失败弹窗 `FailModal`
9. 通关奖励弹窗 `VictoryModal`
10. 收集册页 `Collection`
11. 排行榜页 `Rank`

## 3. 对局页功能规格

## 3.1 区域布局（自上而下）

1. 顶栏（左暂停 + 中倒计时 + 右留空）
2. 棋盘区（分关卡叠牌：普通12列 / 困难20列）
3. 移出区（3 个堆叠位）
4. 卡槽区（7 格单卡槽）
5. 技能栏（移出 / 凑齐 / 洗牌）

- 动物头像与底牌颜色解耦；同局内不同动物底色唯一、同动物同色；跨局可重新分配
- 棋盘规格按关卡切换：普通关 4×3（12列，2层）→ 困难关 4×5（20列，5~6层）
- 移出区默认隐藏，仅在移出技能或槽满复活时显示
- 移出区不允许遮挡棋盘可点击牌区域

## 3.2 组件与位置

| 组件ID | 类型 | 建议位置 | 尺寸建议 | 功能 |
|---|---|---|---|---|
| `battle.btnPause` | Icon按钮 | 左上，`x:24~120, y:24~120` | 96x96 | 打开暂停弹窗 |
| `battle.timer` | 文本 | 顶栏中间，`x:270~480, y:24~96` | 高 56~72 | 显示剩余时间 |
| `battle.topRightBlank` | 占位区 | 右上，`x:630~726, y:24~120` | 96x96 | 右侧留空不放功能 |
| `battle.board` | 容器 | 中上，`x:30~720, y:130~760` | 690x630 | 普通关4×3每列2张；困难关4×5每列5~6张 |
| `battle.holdStacks[3]` | 堆叠位 | `y:780~920` | 单个约140x140 | 移出区，支持多层堆叠，默认隐藏 |
| `battle.slots[7]` | 单卡槽 | `y:920~1085` | 单个约160x160 | 7槽单卡；每次入槽/回槽自动重排，同动物相邻，满3张消除 |
| `battle.btnMoveOut` | 技能Icon | 左下，`x:90~240, y:1120~1260` | 128x128 | 移出技能 |
| `battle.btnComplete` | 技能Icon | 中下，`x:311~439, y:1120~1260` | 128x128 | 凑齐技能 |
| `battle.btnShuffle` | 技能Icon | 右下，`x:510~660, y:1120~1260` | 128x128 | 洗牌技能 |

## 3.3 对局点击规则

| 点击对象 | 是否可点 | 结果 |
|---|---|---|
| 列顶牌 | 可点 | 入7槽中的空槽（单槽单卡） |
| 非顶牌 | 不可点 | 无响应（可给轻震动） |
| 移出区牌 | 可点（可选） | 退回7槽或交换逻辑 |
| 技能Icon（有次数） | 可点 | 执行对应技能 |
| 技能Icon（无次数） | 不可点 | 置灰 + “次数不足”提示 |
| 移出技能执行 | 可点 | 不进入选卡态，按7槽当前重排顺序直接上移前端最多3张到移出区（FIFO） |
| 移出区顶层卡 | 可点 | 回到7槽空位；仅可逐层点击每摞最上层 |

## 4. 弹窗系统规格（全量）

## 4.1 弹窗清单

| 弹窗ID | 触发时机 | 是否冻结时间 | 关闭方式 |
|---|---|---|---|
| `SettingsModal` | 首页点设置 | 否 | 右上 X / 背景点击（可选） |
| `RulesModal` | 首页点规则 | 否 | 右上 X / 背景点击（可选） |
| `InvasionModal` | 普通关清盘后 | 是 | **3秒自动关闭** |
| `PauseModal` | 对局点暂停 | 是 | 继续/退出 |
| `ReviveModal` | 失败（超时/槽满，且该类型复活次数>0） | 是 | 复活方式/放弃 |
| `FailModal` | 放弃或失败类型复活次数已耗尽 | 是 | 再试一次/回首页 |
| `VictoryModal` | 困难关通关 | 是 | 查看收集册/回首页 |

## 4.2 转场弹层（大批量动物来袭）

- 触发：普通关牌清空
- 显示时长：3 秒（3→2→1）
- 行为：倒计时结束后强制进入困难关（用户无其他分支可选）
- 禁止项：
  - 不能手动关闭
  - 无“继续挑战”按钮
  - 无“困难关开启”副标题按钮区
  - 无“返回首页/停留本页/取消进入第二关”入口

## 4.3 暂停弹窗

| 按钮 | 功能 | 跳转 |
|---|---|---|
| 继续 | 关闭弹窗并恢复计时 | 当前对局 |
| 退出 | 结束本局并回首页 | `Home` |

## 4.4 复活弹窗（重点）

- 触发：
  - `timer == 0`（超时）
  - 7 槽被单卡占满（槽满）
- 规则：
  - 复活按失败类型独立计次：超时 1 次 + 槽满 1 次（互不占用）
  - “看视频复活”与“分享复活”按钮样式完全一致，只能图标/文案不同
- 成功后：
  - 超时：直接加时（无额外动画弹层）
  - 槽满：7 槽卡牌全部上移，按 3-3-1 分摞（无额外动画弹层）

## 4.5 胜利判定

- 仅当 **棋盘区 + 7槽 + 移出区** 全部无卡牌时，判定胜利
- 任一处仍有卡牌，均不可触发通关

## 5. 收集册与排行榜规格

## 5.1 收集册页面

- 目标：展示总图鉴与已解锁进度
- 总动物池：评审版建议 **36 种**
- 网格建议：
- 36 格（6 行 x 6 列）
  - 每格 200x200 素材（程序按比例缩放）
- 状态样式：
  - 未获得：黑底白问号
  - 已获得：彩色动物卡点亮

## 5.2 排行榜页面

- 双页签：
  - `tab.totalCaught`：按累计抓到动物总数量排名（抓到几个）
  - `tab.speciesCaught`：按已解锁动物种类数排名（抓到几种，`X/36`）
- 不展示：对局分数、金币、战力
- 基础字段：
  - 昵称
  - 头像
  - 右侧统计值（随页签切换：总数量 或 种类数）
  - 排名

## 7. 功能状态机（高层）

`Home -> NormalBattle -> InvasionModal(3s auto) -> HardBattle -> VictoryModal -> Home`

失败分支：

- `Battle -> ReviveModal(失败类型仍有复活次数)`
- `Revive成功 -> Battle继续`
- `Revive放弃/失败类型次数耗尽 -> FailModal -> (Retry/Home)`

---

## 8. 异常与边界处理

1. 弹窗重复触发保护：同一帧只允许一个高优先级弹窗
2. 倒计时到 0 与点牌同时发生：以“失败判定”优先
3. 技能与复活并发：复活期间锁技能点击
4. 网络失败（广告/分享）：提示后返回复活弹窗，不吞机会
5. 中途切后台：自动暂停并弹出 `PauseModal`
