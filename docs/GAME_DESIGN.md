# 萌宠大冒险 — 玩法功能规格

> 本文档为当前唯一权威的玩法规格。代码实现以本文为准。

## 1. 产品定位

- **名称**：萌宠大冒险
- **平台**：只面向 **微信小程序移动端**（Web 仅作本地原型调试）
- **视觉**：全页面 **3D 萌系**；顶栏/底栏功能以 **Icon 图标** 为主，文字由程序叠加
- **流程**：首页 → 普通关 → 转场（3 秒自动）→ 困难关 → 通关奖励 → 回首页

- 底部三按钮可用 Icon + 程序叠字，或 Icon 为主

## 3. 对局页

### 3.1 布局

```
┌─────────────────────────────────────┐
│ [⏸]            01:23              │  ← 左上暂停 Icon，中间倒计时，右侧留空
├─────────────────────────────────────┤
│                                     │
│ 普通:4×3(12列×2层) / 困难:4×5(20列×5~6层) │  ← 分关卡棋盘规格
│                                     │
├─────────────────────────────────────┤
│ [移出区：默认隐藏，激活后显示 3 个堆叠位] │  ← 不遮挡棋盘（见 §3.3）
├─────────────────────────────────────┤
│[槽1][槽2][槽3][槽4][槽5][槽6][槽7]    │  ← 7 格单卡槽（每槽仅 1 张）
├─────────────────────────────────────┤
│    [移出]      [凑齐]      [洗牌]     │  ← 三技能：仅 Icon，无文字
└─────────────────────────────────────┘
```

| 区域 | 说明 |
|------|------|
| 顶栏左 | **暂停 Icon**，点击打开暂停弹窗 |
| 顶栏中 | **倒计时** `MM:SS`，不显示分数、无关卡号 |
| 顶栏右 | **留空**，不放功能按钮 |
| 棋盘 | 分关卡叠牌：普通 **4×3=12列（每列2层）**；困难 **4×5=20列（每列5~6层）**，仅每列顶牌可点 |
| 移出区 | 位于 7 槽上方，**默认隐藏**；使用移出技能或槽满复活时显示，且不遮挡棋盘可点击牌 |
| 卡槽区 | **7 格单卡槽**，每槽仅放 1 张卡；每次入槽/回槽后自动重排，相同动物相邻聚拢，满3张即消除 |
| 底栏 | **移出 · 凑齐 · 洗牌** 三个 **Icon 按钮** |

### 3.2 动物与底牌颜色规则（解耦）

- 动物头像与牌底色是**分离资产**：
  - 动物头像来自 36 种动物池
  - 牌底色来自独立颜色池
- 对局开始时，为本局激活动物集合做一次颜色映射（animal -> color）
- 规则约束：
  1) **同一局内**：不同动物底色不能重复  
  2) **同一局内**：同一种动物必须保持同一个底色  
  3) **跨局**：同一种动物可以换成新的底色（不要求全局固定）
- 所有牌统一：圆角方形、3D 厚边、柔和阴影；仅底色与头像组合区分

### 3.3 移出区（上移区）

- 位于 **7 槽上方**，视觉上独立一行（3 个堆叠位），默认隐藏
- 出现条件：
  - 使用「移出」技能后显示
  - 槽满触发复活上移后显示
- 不遮挡规则：移出区不得覆盖棋盘可点击区域，不能挡住上方可点卡牌
- 使用移出技能：从当前 7 槽（重排后顺序）前端自动上移 **最多 3 张**（FIFO，不手动选牌；不足 3 张则全上移）
- 槽满复活上移：将 7 槽全部上移到 3 个堆叠位，按层分摞（例如 6 张 = 下层3张 + 上层3张；7 张 = 3-3-1）
- 回放机制：玩家可点击移出区卡牌回到 7 槽，只能点击每摞最上层，逐层取回
- 复活上移后不弹成功页，直接回到对局继续

## 4. 核心机制 — 7 格单卡槽

| 行为 | 规则 |
|------|------|
| 入槽 | 点击列顶牌后，卡牌进入下方 7 槽空位（单槽单卡）并立即触发槽内重排 |
| 槽内重排 | 每次入槽/回槽后，将 7 槽内卡牌重新排列；相同动物必须相邻聚拢 |
| 三消 | 槽内同动物达到 3 张即自动消除（重排后同动物连续展示） |
| 队列顺序 | 7 槽队列以前端到末端的**当前重排结果**为准；移出技能始终处理队列前端 |
| 失败·槽满 | 7 槽被单卡占满且无法继续消除时判负 |
| 失败·超时 | 倒计时归零 |
| 胜利条件 | **棋盘区 + 7 槽 + 移出区** 所有卡牌全部清空才算胜利 |

## 5. 两关制与牌量（普通轻量 / 困难压迫）

### 5.0 分关卡棋盘规格

- **普通关**：4 行 × 3 列 = 12 列；每列固定 2 层  
  - 总牌数固定：`12 * 2 = 24`
- **困难关**：4 行 × 5 列 = 20 列；每列 5~6 层  
  - 总牌数公式：`N = 20*5 + x = 100 + x`（`x` 为 6 层列数量）

### 5.1 可消约束（先定约束再定牌数）

- 全关卡通用：
  1) `N % 3 == 0`  
  2) 每个动物牌数必须是 `3` 的倍数  
  3) 所有动物牌数之和 = `N`
- 困难关（20 列 5~6 层）可用总牌数：`102 / 105 / 108 / 111 / 114 / 117 / 120`

### 5.2 当前推荐牌量与时间（可直接实现）

| 关卡 | 棋盘 | 目标牌数 N | 层高分布 | 建议倒计时 | 超时复活加时 |
|------|------|------------|----------|------------|--------------|
| 普通 | 4×3（12列） | **24** | 全部 2 层 | **120 秒** | +30 秒 |
| 困难 | 4×5（20列） | **117** | 3 列 5 层 + 17 列 6 层 | **480 秒** | +90 秒 |

### 5.3 难度定义（为什么这样配）

- **普通 24 / 120s**：快速热身，避免“开局牌太多”导致疲劳
- **困难 117 / 480s**：保留高压与策略成本，承担核心挑战体验
- 两关在“棋盘尺寸 + 层数 + 牌量”上有明显梯度，体感清晰

### 5.4 动物池规模与单局抽样（正式版）

- 正式动物池：**36 种**
- 单局抽样：
  - 普通关：随机抽 **8 种**
  - 困难关：随机抽 **18 种**
- 收集册进度统一为 **X/36**；排行榜采用双页签（抓到几个 / 抓到几种）

### 5.5 可直接使用的动物配比模板

> 每个数字都为 3 的倍数，可保证理论可消。

**普通关 24 张（推荐）**

- 抽样 8 种后分配：  
  `3, 3, 3, 3, 3, 3, 3, 3`（共 8 种）

**困难关 117 张（推荐）**

- 抽样 18 种后分配：  
  `9, 9, 9, 9, 9, 9, 6, 6, 6, 6, 6, 6, 6, 6, 6, 3, 3, 3`（共 18 种）

### 5.6 关卡生成额外约束（防止“理论可消但体验崩”）

- 普通关：保持轻量，避免把同一种动物全部压在同列底部
- 困难关：
  - 开局前 20 次可点击顶牌中，最多出现 5 种动物，避免开局秒死
  - 同一种动物不要连续压在同列最底层（避免强卡）
  - 允许“陷阱段”，但全局至少保留 1 条可推进链
- 每局生成后做可解性校验，不通过则重排

### 5.7 转场：大批量动物来袭

- 普通关完成后 **全屏转场弹层**
- 主文案：**「大批量动物来袭！」**
- **3 秒倒计时** 大号数字（3 → 2 → 1），**自动进入困难关**
- 普通关结束后只允许进入第二关：不提供继续挑战/返回/关闭等任何分支交互，倒计时结束后强制进入困难关

### 5.8 困难关通关奖励

- 最后消除的动物在屏幕中心 **炫酷奖励弹层**
- 动物从卡牌跳出 → 飞入收集册
- 按钮：查看收集册 / 回到首页

### 5.9 弹层一览

| 场景 | 弹层 | 要点 |
|------|------|------|
| 普通关完成 | 大批量动物来袭 | 3 秒后强制进入困难关（无任何可选分支） |
| 困难关完成 | 通关奖励 | 最后动物 + 收集册 |
| 暂停 | 继续 / 退出 | 时间冻结 |
| 失败（超时/槽满） | 复活选择 | 对应失败类型可触发复活；**两按钮样式相同**，仅 Icon/文案不同 |
| 复活执行 | **无弹层** | 超时 → **加时间**；槽满 → **7槽全上移（3-3-1分摞，支持多层堆叠）** |
| 失败且对应类型复活次数耗尽/放弃 | 最终失败 | 再试一次 / 回首页 |

### 5.10 复活规则

- 复活按失败类型分通道，互不占用：
  - **超时复活**：每局最多 1 次
  - **槽满复活**：每局最多 1 次
- 即：同一局最多可发生 2 次复活（超时一次 + 槽满一次）
- 失败原因与复活效果：

| 失败原因 | 复活效果（无动画弹层，直接生效） |
|----------|----------------------------------|
| **时间到** | 按当前关卡增加时间（普通 +30 秒 / 困难 +90 秒），继续对局 |
| **槽满** | 复活时自动将 7 槽卡牌全部上移到移出区，按层分摞（如 3-3-1） |

- 看视频复活、分享微信复活：**按钮外观完全一致**（同一套 3D 按钮样式），仅图标与文案不同
- 复活入口仅在对应失败类型下可用；该类型复活次数耗尽后，同类型再次失败直接进入最终失败

## 6. 三技能

底栏三个 **Icon**：移出 · 凑齐 · 洗牌

| 技能 | 效果 |
|------|------|
| 移出 | 点击技能后自动从 7 槽（重排后）队列前端取 **最多 3 张** 上移到移出区（FIFO） |
| 凑齐 | 从当前未消牌中补齐一次三消（仅消现有牌，不新增/生成新牌） |
| 洗牌 | 按当前关卡棋盘重分列（普通 12 列 / 困难 20 列） |

次数经济：每日各 1 次免费；用完可看广告/分享补 1 次（每局每技能限 1 次）；耗尽置灰。

## 7. 收集册

| 状态 | 视觉 |
|------|------|
| **未获得** | **黑色卡片**，中央 **白色问号 `?`**，无动物头像 |
| **已获得** | **彩色点亮**，对应动物 3D 卡（收集册展示色可独立配置，不强制与当局分色一致） |

- 共 36 种动物，网格展示，进度 **X/36**
- 困难关通关解锁 1 种（本局奖励动物）

## 8. 排行榜

- 排行榜采用 **双页签**（参考抓大鹅）：
  1) **抓到几个**：按累计抓到的动物总数量排名（可重复累计）
  2) **抓到几种**：按已解锁动物种类数排名（`X/36`）
- 列表字段统一：用户头像、用户名称、右侧对应统计值、排名
- 不展示对局分数、金币、战力

## 10. 一句话定义

> **萌宠大冒险**：微信小程序分关卡叠牌三消：普通关 **4×3、每列2层（24张）**，困难关 **4×5、每列5~6层（推荐117张）**；配合 **7 格单卡槽** 与按需显示的上方移出区（3 堆位，支持多层堆叠）；**36 种动物总池 + 单局随机抽样（普通8/困难18）**；普通 **120 秒** → **3 秒自动转场** → 困难 **480 秒**；复活无成功弹层（按关卡加时 / 槽满全上移）；胜利需棋盘、卡槽、移出区全部清空。
